An improved CSS baseline

Default browser renderings have converged in a way that mostly obsoletes the venerable resets of yore. Consistency does not imply optimality. There is room to improve the defaults without relying on classes or HTML structures particular to any project. The rules here are implemented on rossabaker.com, but could be exported to start new projects.

Variables #

Some values in this reconditioning are configurable, and will be defined as :root: variables with a --cromulent- prefix.

Spacing #

Border box sizing #

border-box sizing includes the padding and margin of a box in its width and height, which can substantially simplify layout calculations. It wasn’t an option before the 2010s, which is perhaps the reason it’s not the global default now. It will be the default here.

MIT

*, *::before, *::after {
    box-sizing: border-box;
}

Body margin #

The default in most browsers is 8px, which is a bit cramped.

MIT

:root {
    --cromulent-body-margin: 1rem;
}

body {
    margin: var(--cromulent-body-margin);
}

Harmonic ratios #

Text looks nice when it’s sized “harmonically,” with each level a multiple of the previous. 5 steps are defined, with a multiplier of \( \sqrt[5]{2} \).

This corresponds to the 5edo musical scale. Give it a listen!

There is a pow() function in CSS, but browser compatibility is not there yet.

MIT

:root {
    --cromulent-step-1: 1.149;
    --cromulent-step-2: 1.320;
    --cromulent-step-3: 1.516;
    --cromulent-step-4: 1.741;
    --cromulent-step-5: 2.000;
}

These steps are used throughout the reconditioning.

Line heights #

WCAG requires support for line spacing up to 1.5 times the font size. The third step of our harmonic progression is just about there.

MIT

:root {
    --cromulent-line-height: var(--cromulent-step-3);
}

body {
    line-height: var(--cromulent-line-height);
}

Block margins #

It also calls for paragraph spacing two times the font size. It’s a touch too much, but the default 1em falls short. It’s nice when paragraphs and headers have the same margin, and one line height looks good.

MIT

h1, h2, h3, h4, h5, h6, p, blockquote {
    margin-block: calc(var(--cromulent-line-height) * 1rem);
}

Headings #

Relative size of headings #

Heading sizes follow the scale, with no header smaller than the base font. The end result is that <h1> retains its default size of double the base.

MIT

h1 { font-size: calc(var(--cromulent-step-5) * 1rem); }
h2 { font-size: calc(var(--cromulent-step-4) * 1rem); }
h3 { font-size: calc(var(--cromulent-step-3) * 1rem); }
h4 { font-size: calc(var(--cromulent-step-2) * 1rem); }
h5 { font-size: calc(var(--cromulent-step-1) * 1rem); }
h6 { font-size: 1rem; }

Balanced text wrapping #

Balanced text wrapping wraps multi-line headers to roughly equal length, rather than dangling the last word or two. Browser support is still modest in late 2023, but we’re ready when they are.

MIT

h1, h2, h3, h4, h5, h6 {
    text-wrap: balance;
}

<hgroup> #

The <hgroup> element is relatively obscure. It contains:

  • zero or more <p> elements
  • a header element, one of <h1> through <h6>
  • zero or more <p> elements

Paragraphs before the header are bizarre, but a single paragraph after the header is a nifty way to render a subtitle.

MIT

hgroup :is(h1, h2, h3, h4, h5, h6) {
    margin-bottom: 0;
}

hgroup :is(h1, h2, h3, h4, h5, h6) ~ p {
    margin-top: 0;
    font-weight: bold;
    font-style: italic;
}

Fonts #

Prefer default system fonts #

System fonts may be “boring,” but they were chosen by better typographers than me, or may have been overridden by users with accessibility needs unknown to me, and that’s good enough for me. This is an adaptation of Bootstrap Reboot’s stack.

MIT

:root {
    --cromulent-system-font:
        system-ui,
        -apple-system,
        "Segoe UI",
        Roboto,
        "Helvetica Neue",
        "Noto Sans",
        "Liberation Sans",
        Arial,
        sans-serif,
        "Apple Color Emoji",
        "Segoe UI Emoji",
        "Segoe UI Symbol",
        "Noto Color Emoji";
}

body {
    font-family: var(--cromulent-system-font);
}

Monospace font #

Several HTML elements, particularly code samples, need a monospace font. The ui-monospace is similar in spirit to system-ui. This rule is adapted from GitHub, presumably one of the more thoughtful font stacks for code anywhere in the browser.

MIT

:root {
    --cromulent-monospace-font:
      ui-monospace,
      SFMono-Regular,
      "SF Mono",
      Menlo,
      Consolas,
      "Liberation Mono",
      monospace
}

code, kbd, pre, samp, var {
    font-family: var(--cromulent-monospace-font);
}

Element tweaks #

<address> #

<address> is traditionally italicized, but, meh.

MIT

address {
    font-style: normal;
}

<blockquote> #

The default margin is expressed in pixels.

MIT

blockquote {
    margin-block-start: calc(var(--cromulent-step-5) * 1rem);
}

<pre> #

Put a scroll bar on preformatted text, but only where necessary.

MIT

pre {
    max-width: 100%;
    overflow-x: auto;
}
